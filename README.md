# Deskemy 简体中文构建

本仓库用于从 [NFRohan/Deskemy](https://github.com/NFRohan/Deskemy) 的官方源码自动构建简体中文版。

它不长期维护一份修改后的 Deskemy 源码，而是在 GitHub Actions 中完成以下操作：

1. 检出指定版本的官方源码；
2. 执行 `scripts/localize-zh.mjs`，替换界面中的英文文本；
3. 运行 Svelte 类型与语法检查；
4. 使用 Tauri 构建 Windows NSIS 安装包；
5. 额外制作带 `.portable` 标记的便携版 ZIP；
6. 输出 `LOCALIZATION_REPORT.md`，列出仍可能显示为英文的文本。

## 构建

打开仓库的 **Actions** 页面，选择 **构建 Deskemy 中文版**，点击 **Run workflow**。

`upstream_ref` 可以填写官方标签、分支或提交，例如：

```text
v1.1.0
```

构建完成后，从该次运行的 **Artifacts** 下载：

- NSIS 安装程序；
- 便携版 ZIP；
- 汉化检查报告。

## 汉化策略

本项目采用“构建时替换固定英文”的方式，不向 Deskemy 本体引入国际化框架，因此改动较小。

替换脚本按文件和上下文处理文字，避免对变量名、类型名等代码内容进行无差别全局替换。官方源码改动后，未命中的规则会记录到 `LOCALIZATION_REPORT.md`，便于后续维护。

## 数据与更新

- 中文构建保留官方应用标识，因此安装版会继续使用原有 Deskemy 的课程库、播放进度、书签和设置。
- 中文构建暂时关闭启动时自动更新，避免被官方英文版覆盖。
- 便携版数据保存在程序目录内的 `data` 文件夹中，与安装版数据分开。

- 2026-07-18：Windows 10 Enterprise LTSC x64 虚拟机，使用 NSIS 安装版完成安装，视频播放正常。
- 便携版尚未完成实际运行测试。

## 上游版本兼容性

工作流可以通过 `upstream_ref` 构建新的官方标签、分支或提交，但汉化规则仍依赖上游文件结构和界面文本。上游更新后可能出现以下情况：

- 源码结构变化导致构建兼容补丁未命中，工作流会失败并输出诊断日志；
- 原英文文案变化导致部分汉化规则未命中，内容会记录到汉化检查报告；
- 新增界面不会自动获得准确中文翻译，需要在构建后检查和补充规则。

因此，新上游版本应先自动构建并生成报告，再经过实际运行检查后发布。

## 已知限制

- Rust 后端直接返回的少量错误信息仍可能是英文。
- 媒体文件自带的音轨、字幕语言名称和章节标题通常不会被翻译；目前仅对播放器中常见的 `Introduction`、`Intro` 和 `Overview` 章节名称做显示层映射。
- 每次官方更新后都应查看汉化检查报告，并实际启动程序核对界面。
